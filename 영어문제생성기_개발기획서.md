# 영어 문제 생성기 — 개발 기획서

> 이 문서는 Claude CLI 또는 AI 에이전트에게 전달하여 개발을 진행하기 위한 기획서입니다.
> 기존 웹사이트에 배너/메뉴를 통해 연결되는 **독립 페이지**로 구현합니다.

---

## 1. 프로젝트 개요

### 목적
영어 단어 목록(영영풀이 포함)과 영어 지문을 입력하면, 수능/내신 스타일의 **영어 어휘 문제를 대량 자동 생성**하고, 문제+해설이 포함된 `.docx` 파일로 다운로드할 수 있는 웹 도구.

### 핵심 흐름
```
[단어 목록 입력] + (선택: 영어 지문 입력) → [AI 문제 생성] → [.docx 다운로드]
```

### 확장 고려사항
- 이 문제 생성기는 **여러 종류의 문제 생성기 중 하나**임
- 향후 추가될 문제 유형: 순서 배열, 문장 삽입, 내용 일치/불일치 등
- 따라서 **공통 프레임워크**(입력 UI, API 호출 로직, docx 출력 파이프라인)를 모듈화해야 함

## 2. 문제 유형 정의

### Type A: 영영풀이 매칭 — 지문 기반

> 입력: 단어 목록 + 영어 지문

지문에서 밑줄 친 단어 2개를 선택하고, 보기 ⓐ~ⓔ에 5개의 영영풀이를 제시(정답 2개 + 오답 3개). 학생이 밑줄 단어에 해당하는 영영풀이 조합을 고르는 5지선다.

#### 예시 출력 형식
```
30. ⓐ~ⓔ 중 글에 나타난 단어에 대한 영영풀이가 아닌 것으로 짝지어진 것은?13)

┌────────────────────────────────────────────────────────┐
│ Do you want to make healthy ramyeon? This is my recipe.│
│ First, boil water and put in ramyeon, sauce, and tteok.│
│ Cut some carrots and gimchi, and add them. Now, this   │
│ is my secret. Put some milk and cheese. It looks tasty,│
│ doesn't it?                                            │
└────────────────────────────────────────────────────────┘

ⓐ long, thin, orange-coloured vegetables
ⓑ to stir or shake two substantes together
ⓒ to generate bubbles of vapor when heated
ⓓ having some heat but not enough to be hot
ⓔ a list of ingredients and a set of instructions
   that tell you how to cook something

① ⓐ ⓑ   ② ⓐ ⓔ   ③ ⓑ ⓒ
④ ⓑ ⓓ   ⑤ ⓓ ⓔ
```

#### 핵심 규칙
- 오답 영영풀이는 지문에 등장하는 다른 단어의 풀이가 아닌 것이어야 함
- 영영풀이는 사용자가 제공한 원본을 그대로 사용
- 5지선다 조합에서 정답이 반드시 하나만 존재

### Type B: 빈칸 영영풀이 매칭 — 문장 생성형

> 입력: 단어 목록만 (지문 불필요 — AI가 예문 자동 생성)

단어 목록에서 4개 단어를 선택하고, 각 단어가 빈칸으로 들어가는 짧은 예문을 AI가 생성. 보기에 6개 영영풀이(정답 4개 + 오답 2개)를 제시. 학생이 (A)~(D) 각 빈칸에 해당하는 영영풀이를 순서대로 맞추는 5지선다.

#### 예시 출력 형식
```
23. 빈칸에 들어갈 단어의 영영 풀이를 <보기>에서 순서대로
    바르게 짝지은 것은?③

ㅇ How long can you (A) on one leg?
ㅇ May I (B) a white wine with this dish?
ㅇ The answer to the question might (C) unclear after the
   discussion.
ㅇ She will (D) him with a delicious homemade cake for
   his birthday.

┌──────────────────────────────────────────────────┐
│ <보기>                                            │
│ ㉠ to change from a solid to a liquid by applying │
│    heat                                           │
│ ㉡ to tell someone your ideas about what they     │
│    should do                                      │
│ ㉢ to be in a steady position without falling to  │
│    one side                                       │
│ ㉣ to continue to exist or be left after others   │
│    have gone                                      │
│ ㉤ to cause someone to feel mild astonishment or  │
│    shock                                          │
│ ㉥ to put it in a position where other people can │
│    see a part of your body                        │
└──────────────────────────────────────────────────┘

     (A)  (B)  (C)  (D)
① ㉢   ㉡   ㉣   ㉤
② ㉢   ㉥   ㉣   ㉠
③ ㉢   ㉥   ㉣   ㉤
④ ㉥   ㉡   ㉠   ㉤
⑤ ㉥   ㉡   ㉣   ㉠
```

#### 핵심 규칙
- AI가 생성하는 예문은 중고등 수준의 자연스러운 영어 (복잡하지 않고, 1~2줄 이내)
- 빈칸 단어는 문맥상 명확하게 유추 가능해야 함
- 오답 영영풀이는 빈칸 중 어느 것과도 매칭되지 않아야 하지만, 얼핏 혼동 가능한 수준
- 선택지에서 정답이 반드시 하나만 존재하도록 조합 구성
- 보기 기호: ㉠㉡㉢㉣㉤㉥ (동그라미 한글 기호)

## 3. ⭐ 품질 기준 (Quality Spec) — 최우선 요구사항

> **문제의 퀄리티가 첨부된 예시 이미지와 실질적으로 동일해야 한다.**
> 아래 기준을 AI 프롬프트, 후처리 검증, docx 렌더링 모든 단계에 적용한다.

### 3.1 영영풀이 품질

| 기준 | 스펙 | 근거 (예시 분석) |
|------|------|-----------------|
| 길이 | 1~2줄 (영문 8~20단어) | 예시: "to tell someone that you are sorry that you have done something wrong" = 15단어 |
| 어휘 수준 | 중학교 수준 영어로 작성 (정의 자체가 쉬워야 함) | 예시에서 정의에 사용된 단어가 모두 기초 어휘 |
| 문장 구조 | "to + 동사 ~" (동사), "the act of ~" (명사), "not ~ / in a way that ~" (형용사/부사) 패턴 | 예시 패턴 그대로 |
| 시작 형태 | 품사별 일관된 시작: v. → "to ~", n. → "a/an/the ~", a. → 형용사적 설명, adv. → "in a way that ~" | 예시 단어 목록에서 확인 |
| 톤 | 사전적, 객관적, 간결 | 교과서/사전 톤 |

**중요**: 사용자가 제공한 영영풀이를 원본 그대로 사용해야 한다. AI가 영영풀이를 수정하거나 재작성하면 안 된다.

### 3.2 예문/문장 품질 (Type B — AI가 생성하는 문장)

| 기준 | 스펙 |
|------|------|
| 길이 | 1줄~최대 2줄 (영문 8~15단어) |
| 어휘 수준 | 중고등 교과서 수준 (불필요하게 어려운 단어 사용 금지) |
| 문맥 명확성 | 빈칸 단어가 문맥상 자연스럽고 유일하게 유추 가능 |
| 문장 유형 다양성 | 평서문, 의문문, 명령문 등 다양하게 섞기 |
| 주어 다양성 | I, you, she, he, they, we, 고유명사 등 다양 |
| 시제 다양성 | 현재, 과거, 미래, 진행 등 다양하게 |
| 자연스러움 | 원어민이 실제로 사용할 법한 자연스러운 문장 |

**Type B 예문 품질 체크리스트** (AI 프롬프트에 포함):
```
✅ 좋은 예: "How long can you (A) on one leg?" — 짧고, 자연스럽고, 빈칸이 명확
✅ 좋은 예: "May I (B) a white wine with this dish?" — 정중한 표현, 실생활 맥락
✅ 좋은 예: "She will (D) him with a delicious homemade cake for his birthday." — 구체적 상황

❌ 나쁜 예: "The person did (A) something." — 너무 모호, 빈칸 유추 불가
❌ 나쁜 예: "The extemporaneous (B) was..." — 불필요하게 어려운 단어
❌ 나쁜 예: "I (A). I (B). I (C). I (D)." — 주어/구조 반복, 단조로움
```

### 3.3 보기(선택지) 품질

| 기준 | 스펙 |
|------|------|
| 오답 매력도 | 오답 풀이가 "얼핏 맞아 보이지만 정확히는 아닌" 수준 (너무 쉬운 오답 금지) |
| 오답 소스 | 같은 단어 목록 내 다른 단어의 영영풀이 우선 사용 → 부족 시 AI 생성 |
| 길이 균형 | 모든 보기의 영영풀이 길이가 비슷해야 함 (정답만 유난히 길거나 짧으면 안됨) |
| 품사 일관성 | 같은 문제 내 보기 풀이는 가급적 같은 품사 패턴 혼재 가능 (예시처럼) |
| 5지선다 조합 | 정답이 ①~⑤ 중 특정 번호에 쏠리지 않도록 랜덤 배치 |

### 3.4 전체 문제 레이아웃 품질

| 기준 | 스펙 |
|------|------|
| 지시문 | 수능/모의고사 스타일 한국어 지시문 (예시와 동일한 어투) |
| 번호 체계 | 문제 번호 + 소문제 없음 (단일 문항 형태) |
| 지문 박스 | Type A: 지문은 테두리 박스 안에 | 
| 문장 리스트 | Type B: 각 문장 앞에 ㅇ 기호, 들여쓰기 |
| 보기 박스 | Type B: \<보기\> 영역은 테두리 박스 안에 |
| 선택지 배열 | Type A: 3개+2개 행 / Type B: (A)(B)(C)(D) 헤더 + 표 형태 |
| 여백/간격 | 문제-보기-선택지 사이 적절한 간격 |

### 3.5 품질 검증 자동화 (후처리)

프로그래밍적으로 반드시 검증할 항목:

```python
# 공통 검증
COMMON_CHECKS = {
    "정답_유일성": "5지선다 중 정답이 정확히 1개인지",
    "보기_수_일치": "보기 개수가 설정값과 일치하는지",
    "영영풀이_원본일치": "사용자 제공 영영풀이와 100% 동일한지 (띄어쓰기, 구두점 포함)",
    "영영풀이_길이": "각 풀이가 8~25단어 범위인지",
    "영영풀이_길이편차": "가장 긴 풀이와 짧은 풀이의 단어 수 차이가 10단어 이내인지",
    "선택지_정답분포": "배치 생성 시 정답 번호가 ①~⑤ 고르게 분포하는지",

}

# Type B 전용 검증
TYPE_B_CHECKS = {

    "빈칸_수_일치": "빈칸 수가 설정값과 일치하는지",
    "예문_자연스러움": "빈칸 단어가 문맥상 유일하게 결정되는지 (다의성 검사)",
}
```

### 3.6 품질 보장을 위한 AI 프롬프트 전략

1. **Few-shot 예시 필수 삽입**: 프롬프트에 위 예시(Type A 30번, Type B 23번)를 그대로 포함하여 출력 형식과 톤을 학습시킴
2. **Negative 예시 포함**: "이렇게 하면 안 된다"는 예시도 프롬프트에 명시
3. **Self-check 지시**: AI에게 생성 후 스스로 품질 기준 체크하고, 기준 미달 시 재생성하도록 지시
4. **Temperature 0.3 이하**: 일관성 있는 출력을 위해 낮은 temperature 유지
5. **후처리 검증 + 재시도**: 프로그래밍적 검증 실패 시 최대 3회 재생성

## 4. 입력 설계

### 4.1 단어 목록 입력

**방식: 구조화된 텍스트 입력 (직접 입력 또는 파일 업로드)**

엑셀은 사용하지 않는다. 대신 다음 방식을 지원한다:

#### Option A: 직접 텍스트 입력 (textarea)
정해진 구분자 형식으로 입력한다. 각 줄이 하나의 단어 엔트리:
```
단어 | 품사 | 한국어뜻 | 영영풀이 | 예문(선택)
```
예시:
```
apologize | v. | 사과하다 | to tell someone that you are sorry that you have done something wrong | She apologized for her mistake.
arrival | n. | 도착 | the act of arriving somewhere | The arrival of the train was on time.
capital | n. | 수도 | an important city that is the center of government of a country |
casual | a. | 평상시의 | not formal or not for a formal situation | We wear casual clothes on special days at school.
communicate | v. | 의사소통을 하다 | to share information with others by speaking, writing, moving your body, etc. |
```

#### Option B: 텍스트 파일 업로드 (.txt, .csv, .md)
위와 동일한 형식의 파일을 업로드한다. 파서가 `|` 구분자로 파싱.

#### Option C: AI 자동 파싱 (보조 기능)
- 비정형 텍스트(교과서 캡처 텍스트, 자유 형식 단어장)를 붙여넣으면
- Claude API로 구조화된 형식으로 변환 후 사용자에게 확인 요청
- **주의**: 보조적 기능이며, 파싱 결과를 반드시 사용자가 확인/수정한 후 진행

> **설계 원칙**: AI가 영영풀이를 새로 "만드는" 것이 아니라, 사용자가 제공한 영영풀이를 그대로 사용해야 한다. 문제의 신뢰성과 교육적 정확성을 위해 영영풀이는 사용자 입력을 원본으로 취급.

### 4.2 영어 지문 입력 (Type A 전용)

- **textarea**로 직접 입력
- 여러 지문을 한 번에 입력 가능 (구분선 `---` 또는 번호로 구분)
- 각 지문에 대해 독립적으로 문제가 생성됨
- 시스템이 자동으로 지문 내 등장 단어를 단어 목록과 매칭

### 4.3 문제 유형별 입력 요구사항 요약

| | Type A (지문 기반) | Type B (문장 생성형) |
|---|---|---|
| 단어 목록 | ✅ 필수 | ✅ 필수 |
| 영어 지문 | ✅ 필수 | ❌ 불필요 (AI가 문장 생성) |
| 최소 단어 수 | 5개 이상 (정답2 + 오답3) | 6개 이상 (정답4 + 오답2) |

### 4.4 문제 생성 옵션

#### Type A 옵션
| 옵션 | 설명 | 기본값 |
|------|------|--------|
| 문제 번호 시작 | 문제 번호 시작점 | 1 |
| 지문당 문제 수 | 한 지문에서 몇 문제 생성 | 1 |
| 보기 수 | 영영풀이 보기 개수 (ⓐ~ⓔ) | 5 |
| 밑줄 단어 수 | 정답 단어/풀이 수 | 2 |
| 오답 풀이 소스 | 같은 단어 목록에서 / AI 생성 | 단어 목록 |

#### Type B 옵션
| 옵션 | 설명 | 기본값 |
|------|------|--------|
| 문제 번호 시작 | 문제 번호 시작점 | 1 |
| 빈칸 문장 수 | (A)~(D) 등 빈칸 문장 수 | 4 |
| 보기 풀이 수 | ㉠~㉥ 등 영영풀이 보기 수 | 6 |
| 오답 풀이 수 | 보기 풀이 수 - 빈칸 문장 수 | 2 |
| 생성할 문제 수 | 한 번에 몇 문제 생성 | 5 |

## 5. AI 문제 생성 로직

### 5.1 Type A 파이프라인

```
Step 1: 입력 검증
  - 단어 목록 파싱 & 유효성 검사
  - 지문 텍스트 검증
  - 지문 내 단어 매칭 확인 (최소 2개 이상 매칭 필요)

Step 2: 문제 구성 요소 결정 (프로그래밍)
  - 지문에서 밑줄 칠 단어 2개 선택
  - 오답 영영풀이 3개 선택 (단어 목록 내 지문 미등장 단어에서)

Step 3: Claude API 호출
  - 해설 생성
  - 문제 포맷 최종 확인

Step 4: 후처리 검증 → docx 생성
```

### 5.3 Type A — Claude API 프롬프트

```
너는 한국 중고등학교 영어 시험 문제 출제 전문가야.
주어진 단어 목록과 영어 지문을 바탕으로 영영풀이 매칭 문제를 출제해.

## 참고 예시 (이 형식과 품질 수준을 정확히 따를 것)
---
30. ⓐ~ⓔ 중 글에 나타난 단어에 대한 영영풀이가 아닌 것으로 짝지어진 것은?

[지문]
Do you want to make healthy ramyeon? This is my recipe. First, boil water and put in
ramyeon, sauce, and tteok. Cut some carrots and gimchi, and add them. Now, this is my
secret. Put some milk and cheese. It looks tasty, doesn't it?

ⓐ long, thin, orange-coloured vegetables
ⓑ to stir or shake two substantes together
ⓒ to generate bubbles of vapor when heated
ⓓ having some heat but not enough to be hot
ⓔ a list of ingredients and a set of instructions that tell you how to cook something

① ⓐ ⓑ   ② ⓐ ⓔ   ③ ⓑ ⓒ
④ ⓑ ⓓ   ⑤ ⓓ ⓔ
---

## 규칙
1. 영영풀이는 사용자가 제공한 원본을 한 글자도 바꾸지 말고 그대로 사용
2. 오답 영영풀이는 지문에 등장하지 않는 단어의 풀이를 사용
3. 5지선다 조합은 정답이 정확히 1개만 되도록 구성
4. 해설: 각 밑줄 단어↔풀이 매칭 이유 + 오답 풀이가 왜 무관한지 설명
5. 모든 보기 풀이의 길이가 비슷하도록 구성 (정답만 유난히 길거나 짧으면 안됨)

## 자기 점검 (생성 후 반드시 확인)
- 정답 조합이 정확히 1개인가?
- 영영풀이가 원본과 100% 동일한가?
- 보기 풀이 길이가 비슷한가?

## 출력: 반드시 아래 JSON만 (마크다운 백틱 없이)
{
  "question_number": number,
  "instruction": "ⓐ~ⓔ 중 글에 나타난 단어에 대한 영영풀이가 아닌 것으로 짝지어진 것은?",
  "passage": "지문 전체 텍스트",
  "underlined_words": [
    {"word": "단어", "definition_label": "ⓐ"}
  ],
  "options": [
    {"label": "ⓐ", "definition": "영영풀이 원문 그대로", "source_word": "원본단어", "is_correct_match": true}
  ],
  "choices": [
    {"number": "①", "combination": ["ⓐ", "ⓑ"], "is_correct": false}
  ],
  "answer": "③",
  "explanation": "해설 텍스트"
}
```

### 5.4 Type B — Claude API 프롬프트

```
너는 한국 중고등학교 영어 시험 문제 출제 전문가야.
주어진 단어와 영영풀이를 바탕으로, 빈칸 영영풀이 매칭 문제를 출제해.

## 참고 예시 (이 형식과 품질 수준을 정확히 따를 것)
---
23. 빈칸에 들어갈 단어의 영영 풀이를 <보기>에서 순서대로 바르게 짝지은 것은?

ㅇ How long can you (A) on one leg?
ㅇ May I (B) a white wine with this dish?
ㅇ The answer to the question might (C) unclear after the discussion.
ㅇ She will (D) him with a delicious homemade cake for his birthday.

<보기>
㉠ to change from a solid to a liquid by applying heat
㉡ to tell someone your ideas about what they should do
㉢ to be in a steady position without falling to one side
㉣ to continue to exist or be left after others have gone
㉤ to cause someone to feel mild astonishment or shock
㉥ to put it in a position where other people can see a part of your body

     (A)  (B)  (C)  (D)
① ㉢   ㉡   ㉣   ㉤
② ㉢   ㉥   ㉣   ㉠
③ ㉢   ㉥   ㉣   ㉤
④ ㉥   ㉡   ㉠   ㉤
⑤ ㉥   ㉡   ㉣   ㉠
---

## 문장 생성 규칙 (매우 중요 — 퀄리티 핵심)
1. 각 문장은 8~15단어, 1~2줄 이내
2. 중고등 교과서 수준 어휘만 사용 (빈칸 단어 외에 어려운 단어 금지)
3. 빈칸 단어가 문맥상 자연스럽고 명확하게 유추 가능해야 함
4. 4개 문장의 주어를 최소 3종류 이상 다양하게
5. 문장 유형 다양: 평서문, 의문문, 명령문 등 섞기
6. 시제 다양: 현재, 과거, 미래, 진행 등 섞기
7. 원어민이 실제로 사용할 법한 자연스러운 문장
8. 예시의 문장 길이와 복잡도를 참고하여 동일한 수준으로 생성

## 좋은 문장 vs 나쁜 문장
✅ "How long can you (A) on one leg?" — 짧고, 자연스럽고, 빈칸이 명확
✅ "May I (B) a white wine with this dish?" — 정중한 표현, 실생활 맥락
✅ "She will (D) him with a delicious homemade cake for his birthday." — 구체적 상황
❌ "The person did (A) something." — 너무 모호, 빈칸 유추 불가
❌ "The extemporaneous (B) was..." — 불필요하게 어려운 단어
❌ "I (A). I (B). I (C). I (D)." — 주어/구조 반복, 단조로움

## 보기 규칙
1. 영영풀이는 사용자 제공 원본을 그대로 사용 (한 글자도 변경 금지)
2. 오답 풀이도 사용자 단어 목록에서 가져옴
3. 모든 보기 풀이 길이가 비슷해야 함
4. 오답 풀이는 빈칸 단어 중 어느 것과도 매칭 안 되지만, 얼핏 혼동 가능한 수준

## 선택지 규칙
1. 5지선다에서 정답이 정확히 1개
2. 오답 조합은 1~2개만 틀리게 (전부 틀린 조합은 매력 없음)
3. 정답 위치는 ①~⑤ 중 랜덤

## 자기 점검 (생성 후 반드시 확인)
- 각 문장의 빈칸 단어가 문맥상 유일하게 결정되는가?
- 4개 문장의 주어가 최소 3종류인가?
- 문장 길이가 8~15단어인가?
- 영영풀이가 원본과 100% 동일한가?
- 보기 풀이 길이가 비슷한가?
- 정답 조합이 정확히 1개인가?

## 출력: 반드시 아래 JSON만 (마크다운 백틱 없이)
{
  "question_number": number,
  "instruction": "빈칸에 들어갈 단어의 영영 풀이를 <보기>에서 순서대로 바르게 짝지은 것은?",
  "sentences": [
    {"label": "(A)", "sentence": "How long can you (A) on one leg?", "answer_word": "stand"}
  ],
  "options": [
    {"label": "㉠", "definition": "영영풀이 원문 그대로", "source_word": "원본단어", "matches_blank": null}
  ],
  "option_mapping": {"(A)": "㉢", "(B)": "㉥", "(C)": "㉣", "(D)": "㉤"},
  "choices": [
    {"number": "①", "mapping": {"(A)": "㉢", "(B)": "㉡", "(C)": "㉣", "(D)": "㉤"}, "is_correct": false}
  ],
  "answer": "③",
  "explanation": "해설 텍스트"
}
```

## 11. Claude CLI 개발 프롬프트

### 프롬프트 1: 기존 웹사이트에 문제 생성기 메뉴 연결

```
이 프로젝트의 웹사이트 구조를 분석해줘.
기존 네비게이션/배너에 "문제 생성기"라는 메뉴 항목을 추가해줘.
이 메뉴를 클릭하면 하위 메뉴가 나타나야 해:
  - 영영풀이 매칭 (지문 기반)
  - 빈칸 영영풀이 매칭 (문장 생성형)
각 하위 메뉴는 별도 페이지로 연결.
기존 디자인 시스템과 일관되게 만들어줘.
```

### 프롬프트 2: Type A + Type B 핵심 개발

```
영어문제생성기_개발기획서.md 파일을 참고해서 영어 문제 생성기를 개발해줘.

지금 구현할 문제 유형 2개:
- Type A: 영영풀이 매칭 (지문 기반) — 기획서 섹션 2의 Type A, 프롬프트는 섹션 5.3
- Type B: 빈칸 영영풀이 매칭 (문장 생성형) — 기획서 섹션 2의 Type B, 프롬프트는 섹션 5.4

구현 순서:
1. common/ 공통 모듈 (input_parser, api_client, quality_checker, docx_generator)
2. types/eng_def_passage/ (Type A 전용 로직)
3. types/eng_def_blank/ (Type B 전용 로직)
4. Streamlit UI (app.py + 각 타입별 ui_page.py)

⭐ 품질이 최우선: 기획서 섹션 3 "품질 기준"을 반드시 읽고,
   모든 AI 프롬프트와 후처리 검증에 해당 기준을 적용해줘.
   특히 생성되는 문장/예문의 길이, 수준, 자연스러움이 예시와 동일해야 함.

기획서의 "8.1 모듈 구조"를 따르고,
향후 다른 문제 유형을 추가할 수 있도록 모듈화해줘.
```

### 프롬프트 3: 추가 문제 유형 개발 시 (템플릿)

```
영어문제생성기_개발기획서.md를 참고해줘.
기존 common/ 모듈과 types/ 하위 구현을 참고해서
새로운 문제 유형 "[문제유형명]"을 types/[폴더명]/에 추가해줘.
공통 모듈은 재사용하고, 문제 유형 전용 로직만 새로 구현해.
app.py 라우팅에도 새 페이지를 추가해줘.
기획서 섹션 3의 품질 기준을 이 유형에도 동일하게 적용해.
```
